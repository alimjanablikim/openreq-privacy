 # OpenReq Privacy Policy

  Last updated: May 14, 2026

  OpenReq is a local API client for creating, saving, and sending HTTP requests from your device.

  ## Data OpenReq Stores

  OpenReq stores app data locally on your device, including workspaces, request tabs, request URLs, headers, parameters, bodies, authentication fields, environment
  variables, response history, and app settings.

  Native builds store this data in local JSON files. The web build stores this data in browser LocalStorage.

  ## Data OpenReq Sends

  OpenReq sends HTTP requests only when you choose to send a request. Request data is sent to the URL you enter or import into the app. Those destinations are
  controlled by you, not by OpenReq.

  OpenReq does not send your workspaces, history, settings, authentication values, or response bodies to an OpenReq server.

  ## Accounts and Cloud Sync

  OpenReq does not include an OpenReq account system, cloud sync, analytics, advertising, or third-party tracking.

  ## Third-Party Services

  If you send a request to a third-party API, that API provider may receive and process the request data you send. Review that provider's privacy policy and terms
  before sending sensitive data.

  OAuth authorization flows may open the authorization URL configured by you. OpenReq receives only the callback result needed to continue the local request flow.

  ## Device Permissions

  OpenReq uses network access to send HTTP requests. It does not request access to camera, microphone, contacts, photos, location, Bluetooth, or health data.

  ## Data Deletion

  You can delete local data from OpenReq settings using the local history and local data reset controls. You can also remove app data by uninstalling the app or
  clearing site data in the browser version.
